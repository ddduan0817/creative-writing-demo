# 【PRD】创意写作沉浸式工作台

## 1. 背景与目标

### 1.1 背景

文心一言正在构建面向创作者的 AI 原生写作平台。不同于传统文档编辑器"打开空白页 → 自行写作"的模式，创意写作沉浸式工作台通过 **AI Agent 引导式创作**，将创作过程结构化为"灵感激发 → 设定构建 → 世界观搭建 → 角色塑造 → 大纲规划 → 正文生成 → 精修编辑"的完整链路，降低创作门槛，提升创作效率。

工作台覆盖 **5 大写作场景**（小说创作、剧本脚本、电商带货、知识专栏、通用写作），针对不同场景提供差异化的创作流程、AI 引导策略和编辑器形态，满足从文学创作到商业内容的全品类写作需求。

### 1.2 目标

- 提供 5 个写作场景的差异化工作台形态，覆盖文学创作、商业内容、知识生产三大方向
- 实现 Agent 模式下"灵感 → 设定 → 世界观 → 角色 → 大纲 → 正文"的分阶段 AI 引导创作链路
- 实现 Workflow 模式下的传统三栏编辑器 + AI 自由对话，满足有经验创作者的自由度需求
- 提供编辑器内的划词 AI 调整能力（润色/丰富/缩写/改写/氛围增强），支撑正文精修环节
- 构建统一的创作首页，聚合场景入口、创作统计和作品管理

### 1.3 产品定位

| 维度 | 说明 |
|------|------|
| 产品名称 | 文心一言 · 创意写作沉浸式工作台 |
| 目标用户 | 网文作者、短剧编剧、电商内容创作者、知识博主、通用文案工作者 |
| 核心价值 | AI 引导降低创作门槛 + 场景化工作台提升创作效率 + 编辑器内 AI 精修提升内容质量 |
| 技术架构 | Next.js 14 App Router · Zustand 状态管理 · TipTap 编辑器 · Tailwind CSS |

## 3. 首页设计

### 3.1 页面结构

首页作为创作入口，承担场景导航、创作统计、作品管理三大职能：

```
┌──────────┬─────────────────────────────────────────┐
│ 侧边栏    │                                         │
│          │  顶部操作栏（每日任务 · 下载应用 · 通知）    │
│ 创意写作  │                                         │
│ 阅读分析  │  Hero 标题：「高效解决写作问题」           │
│ 更多...   │                                         │
│          │  智能输入框（附件 + 语音 + 发送）            │
│          │                                         │
│          │  场景卡片（5 个）                         │
│          │                                         │
│          │  创作情况 ｜ 近期作品                      │
│          │                                         │
└──────────┴─────────────────────────────────────────┘
```

### 3.2 场景卡片

5 个场景卡片横向排列，顺序为：电商带货 → 知识专栏 → 小说创作 → 剧本脚本 → 通用写作。

**卡片结构：**

```
┌─────────────────┐
│ [icon] 场景名称   │  ← icon 左侧 + 标题右侧，同一行
│                  │
│ 子功能描述文字     │  ← 灰色小字，展示该场景涵盖的子功能
└─────────────────┘
```

**各卡片内容：**

| 场景 | icon | 主题色 | 子功能描述 |
|------|------|--------|-----------|
| 电商带货 | 🛍️ | pink | 直播文本 · 产品带货 · 短视频文案 |
| 知识专栏 | 📚 | emerald | 深度拆书 · 内容解读 · 知识播客 |
| 小说创作 | 📖 | purple | 长篇小说 · 短篇故事 · 网络文学 |
| 剧本脚本 | 🎬 | blue | 短剧剧本 · 漫改脚本 · 分镜设计 |
| 通用写作 | ✏️ | orange | 文章撰写 · 文案润色 · 素材整理 |

**交互：**
- hover：卡片上浮 + 阴影加深
- click：进入对应场景的工作台

### 3.3 创作情况卡片

展示用户的创作成就数据：
- 累计字数（及对标文案，如"相当于一部长篇小说"）
- 连续创作天数
- 作品数

### 3.4 近期作品

采用翻页展示，每页 4 条作品。每条包含：
- 场景图标（左侧，按场景类型展示对应 icon）
- 作品名称（大字）
- 字数 + 更新时间（灰色小字，如"12,340字  更新于03-06 14:30"）
- 「⋮」更多操作按钮（右侧），下拉菜单包含：
  - 复制链接
  - 导出作品
  - 删除（红色文字）

点击作品行可直接进入对应工作台继续创作。

## 4. 工作台类型

### 4.1 类型一：Agent 模式 · 双屏工作台

**适用场景：** 电商带货、知识专栏（直接进入），小说创作、剧本脚本（用户选择 Agent 模式后进入）

Agent 模式是工作台的核心创作形态。通过 AI Agent 在右侧对话面板中引导用户逐阶段完成创作，左侧编辑器同步展示当前阶段的成果。

**布局结构：**

```
┌──────────────────────────────┬──┬───────────────────────────────┐
│                              │  │                               │
│        编辑器（左半屏）        │进│         Chat 面板（右半屏）      │
│                              │度│                               │
│   · 顶部信息栏 + 工具栏       │条│   · AI 对话消息流               │
│   · 阶段内容展示（只读）       │  │   · 灵感卡片选择                │
│   · 正文编辑（可编辑）         │  │   · 设定/世界观/角色/大纲确认卡  │
│   · 划词 AI 调整              │  │   · 用户输入框 + 附件上传        │
│                              │  │                               │
└──────────────────────────────┴──┴───────────────────────────────┘
                                ↑
                           创作进度条（垂直）
```

**创作进度条**位于编辑器和 Chat 面板之间，垂直排列 5 个阶段节点，标签按场景差异化：

| 阶段 | 小说/剧本 | 电商带货 | 知识专栏 |
|------|----------|--------|--------|
| 0 | 设定 | 产品 | 导入 |
| 1 | 世界观 | 策略 | 设定 |
| 2 | 角色 | 故事线 | 角色 |
| 3 | 大纲 | 分幕 | 结构 |
| 4 | 正文 | 精修 | 报告 |

### 4.2 类型二：Workflow 模式 · 三栏工作台

**适用场景：** 小说创作、剧本脚本（用户选择 Workflow 模式后进入）

Workflow 模式面向有明确创作思路的用户，提供自由度更高的编辑环境。左栏管理作品设定和结构，中栏为富文本编辑器，右栏为 AI 自由对话（无阶段流程约束）。

**布局结构：**

```
┌─────────────┬──────────────────────────┬───────────────┐
│   左栏 280px  │      编辑器（中栏）        │  右栏 360px    │
│              │      自适应宽度            │               │
│  作品设定     │   工具栏                  │  AI 自由对话   │
│  风格标签     │                          │  随时呼唤 AI   │
│  角色管理     │   正文编辑区              │  无阶段约束    │
│  大纲管理     │                          │               │
│  章节列表     │                          │               │
└─────────────┴──────────────────────────┴───────────────┘
```

### 4.3 类型三：通用写作 · 模板驱动双屏

**适用场景：** 通用写作

通用写作场景以模板为驱动，用户从左栏选择模板并填写配置项，右栏编辑器生成和编辑内容。

**布局结构：**

```
┌─────────────┬──────────────────────────┐
│   左栏        │      编辑器（右栏）        │
│              │                          │
│  模板分类     │   工具栏                  │
│  模板列表     │                          │
│  表单配置     │   正文编辑区              │
│              │                          │
└─────────────┴──────────────────────────┘
```

## 5. Agent 模式详细设计

### 5.1 模式选择入口

不同场景进入工作台时的行为差异：

| 场景 | 进入行为 |
|------|---------|
| 电商带货、知识专栏 | **跳过模式选择**，直接进入 Agent 模式（用户目标明确，不需要自由编辑入口） |
| 小说创作、剧本脚本 | 展示模式选择界面，用户在 Agent 模式和 Workflow 模式间选择 |

**模式选择界面：**

```
┌──────────────────┐  ┌──────────────────┐
│   Agent 模式      │  │  Workflow 模式     │
│                  │  │                  │
│  AI 一步步引导     │  │  自由编辑          │
│  从灵感到成稿      │  │  随时呼唤 AI       │
│  适合从零开始      │  │  适合有想法的创作者  │
└──────────────────┘  └──────────────────┘
```

### 5.2 剧本子类型选择

剧本脚本进入 Agent 模式后，需先选择具体的剧本子类型：

| 选项 | 说明 |
|------|------|
| 短剧 · 剧本 | 影视短剧的文学剧本/拍摄本 |
| 短剧 · 脚本 | 快节奏卡点本，适合竖屏短剧 |
| 漫剧 · 剧本 | 动态漫画的故事文本 |
| 漫剧 · 脚本 | 为画师提供分格/分页指令 |

### 5.3 Agent 对话消息类型

Agent 模式下的 Chat 面板支持以下消息类型：

| 消息类型 | 发送方 | 说明 |
|---------|--------|------|
| `welcome` | AI | 场景欢迎语，引导用户开始创作 |
| `inspiration` | AI | 灵感卡片（每轮 3 张选项卡供用户选择） |
| `micro-adjust` | AI | 微调提示（"想调整什么？"，用户可回复或跳过） |
| `card-selection` | 用户 | 用户选中的灵感卡片内容 |
| `text` | 双方 | 自由文本输入/回复 |
| `thinking` | AI | 思考中状态（生成内容时的等待态） |
| `settings-card` | AI | 设定确认卡片（结构化展示创作设定） |
| `worldbuilding-card` | AI | 世界观确认卡片 |
| `character-card` | AI | 角色确认卡片 |
| `outline-card` | AI | 大纲确认卡片 |
| `stage-intro` | AI | 阶段过渡提示（进入下一创作阶段时） |
| `subtype-select` | AI | 剧本子类型选择（仅剧本场景） |

### 5.4 Agent 创作流程

Agent 模式将创作过程分为 6 个阶段，每个阶段通过"灵感卡片选择 + 确认卡片"的交互模式推进：

#### 阶段 0：灵感收集

通过 3 轮灵感卡片交互，帮助用户从模糊的想法中锚定创作方向：

```
AI: 场景欢迎语
AI: 第 1 轮灵感卡片（3 张方向选择）
     ↓ 用户选择卡片 / 自由输入
AI: 微调提示（"想调整什么？"）
     ↓ 用户回复微调 / 跳过
AI: 第 2 轮灵感卡片（3 张）
     ↓ ...（同上）
AI: 第 3 轮灵感卡片（3 张）
     ↓ ...（同上）
```

**灵感卡片组件结构：**

```
┌───────────────────────────────────────┐
│  卡片描述文本                           │
│  （2-3 行说明具体创作方向）              │
│                                       │
│  [关键词1] [关键词2] [关键词3] [关键词4]  │  ← pill 标签样式
└───────────────────────────────────────┘
```

**各场景灵感卡片方向差异化：**

| 场景 | 第 1 轮方向 | 第 2 轮方向 | 第 3 轮方向 |
|------|-----------|-----------|-----------|
| 小说创作 | 故事类型（言情/悬疑/奇幻） | 叙事风格（慢节奏/快节奏/双视角） | 故事舞台（古镇/都市/海岛） |
| 电商带货 | 产品主打方向（隐形/防水/场景覆盖） | 视频风格（UGC/特写/剧情反转） | 转化策略（价格锚定/功能叠加/社会证明） |
| 知识专栏 | 分析方向（剧情/设定/角色） | 分析深度（概览/深度/研究） | 输出形式（报告/问答/图谱） |

#### 阶段 1：设定确认

灵感收集完成后，AI 生成结构化的设定确认卡片，同时左侧编辑器展示设定全文。

**设定确认卡片结构（按场景差异化）：**

| 场景 | 设定分组 |
|------|---------|
| 小说/剧本 | 故事概念（核心设定/故事基调/故事走向/核心冲突）+ 写作要素（受众/题材/时空/剧情元素/人物关系/风格调性/结局）+ 写作方式（叙事视角/叙事结构/文风） |
| 电商带货 | 产品信息（产品名称/核心卖点/价格/目标人群）+ 视频策略（时长/风格/平台/语言）+ 角色与转化（出镜角色/开篇策略/转化策略/CTA） |
| 知识专栏 | 书籍信息（书名/作者/总字数/章节数）+ 分析配置（分析方向/分析深度/输出形式）+ 自动识别（核心角色/主要势力/核心设定/预处理状态） |

用户确认设定后，进入下一阶段。

#### 阶段 2：世界观构建

与灵感收集相同的 3 轮卡片交互模式，聚焦于故事世界/视频策略/知识体系的深入设计。完成后展示世界观确认卡片。

**世界观确认内容（按场景差异化）：**

| 场景 | 展示内容 |
|------|---------|
| 小说 | 故事世界（世界概述/时间线）+ 核心场景（5 个场景描述）+ 社会生态 + 隐藏线索 |
| 电商带货 | 故事线概述 + 视频时间线 + 分幕场景（6 个）+ 平台投放策略 + 关键洞察 |
| 知识专栏 | 设定体系总结 + 设定揭露节奏 + 核心设定（5 个）+ 分析特征 + 隐藏线索 |

#### 阶段 3：角色设计

3 轮卡片交互 + 角色确认卡片。

**角色确认卡片结构（按场景差异化）：**

| 场景 | 主角 A | 主角 B | 配角/辅助元素 | 关系网络 |
|------|--------|--------|-------------|---------|
| 小说 | 女主角档案（姓名/身份/性格/外貌/习惯/秘密） | 男主角档案（同上） | 4 个关键配角 | 人物关系描述 |
| 电商带货 | 出镜者档案（身份/表现风格/外形/表演要点/核心策略） | 产品档案（产品名/调性/外观/出场方式/画面技巧） | 字幕系统/BGM/场景道具 | 元素间协作关系 |
| 知识专栏 | 主角档案（身份/性格/外貌/习惯/秘密） | 反派/对照角色档案（同上） | 重要配角列表 | 角色关系描述 |

#### 阶段 4：大纲生成

AI 基于前序阶段的设定、世界观、角色，生成完整的大纲/分幕结构。

**大纲结构（按场景差异化）：**

| 场景 | 结构描述 | 章节/分幕数 | 预估篇幅 |
|------|---------|-----------|---------|
| 小说 | 多幕式章回结构 | 12-20 章 | 8-15 万字 |
| 电商带货 | 钩子→痛点→功能演示→转化收口 | 4-8 个分幕 | 30-90 秒 |
| 知识专栏 | 总览→设定→角色→结构→深度 | 5-10 篇分析 | 每篇 2000-3000 字 |
| 剧本脚本 | 三幕式/场次结构 | 按集数和场次 | 按集数计算 |

#### 阶段 5：正文生成

大纲确认后，进入逐章/逐幕生成模式：

1. 系统基于大纲初始化章节列表
2. 逐章流式生成正文（打字机效果）
3. 生成完成后，编辑器切换为可编辑模式
4. 左侧显示章节目录（可折叠 TOC），支持章节间切换

### 5.5 编辑器分阶段展示

Agent 模式下，左侧编辑器区域根据当前创作阶段（`creationStage`）展示对应的只读内容，让用户随时看到已完成阶段的成果：

| 创作阶段 | 小说/剧本 | 电商带货 | 知识专栏 |
|---------|---------|--------|--------|
| 阶段 1（设定） | 创作设定（故事概念 + 写作要素 + 写作方式） | 视频策略 Brief（产品信息 + 视频策略 + 角色与转化） | 分析配置（书籍信息 + 分析配置 + 自动识别） |
| 阶段 2（世界观） | 世界观（故事世界 + 核心场景 + 社会生态 + 隐藏线索） | 故事线设计（视频概述 + 时间线 + 分幕场景 + 平台策略 + 关键洞察） | 设定体系拆解（体系概述 + 揭露节奏 + 核心设定 + 分析特征 + 隐藏线索） |
| 阶段 3（角色） | 角色档案（女主/男主档案 + 配角 + 人物关系） | 出镜角色设定（出镜者/产品档案 + 辅助元素 + 协作关系） | 角色分析（主角/反派档案 + 配角 + 角色关系） |

## 7. 编辑器功能

| 查看资料 | 弹窗展示已生成的设定/世界观/角色/大纲（按阶段逐步解锁） |
| 历史记录 | 详见 7.4 历史记录功能 |

### 7.2 工具栏

WPS 风格的富文本工具栏：
- 撤销/重做
- 标题级别选择（H1/H2/H3/正文）
- 文字样式：加粗 **B** · 斜体 *I* · 下划线 U · 删除线 ~~S~~

### 7.3 划词 AI 调整

用户选中文字后，在选区下方弹出浮动 AI 工具条，提供 5 种快捷调整能力：

```
┌────────────────────────────┐
│  输入自定义改写要求   [发送]  │
├────────────────────────────┤
│  ✨ 润色一下   · 语言更优美    │
│  🌟 丰富一下   · 增加细节描写  │
│  ↔️ 写短一下   · 精简压缩      │
│  🔄 继续写点   · 向下续写      │
│  🎨 氛围增强   · 增强环境氛围  │
└────────────────────────────┘
```

**交互流程：**

1. 选中文本 → 弹出浮动工具条
2. 点击功能项 / 输入自定义要求 → 弹出轻量对话弹窗，流式展示 AI 生成结果
3. 生成完成 → 底部操作区：**替换原文** / **插入** / **重新生成**
4. 点击"替换原文" → 编辑器内容自动更新，弹窗关闭

**补充说明（参考 AI 编辑器优化方案）：**

- 划词后弹出的是**轻量对话弹窗**（非直接菜单），支持在弹窗内输入自定义指令，输入内容不在右侧对话框上屏
- 弹窗下方同时展示快捷指令菜单（润色/丰富/写短/续写/氛围增强）
- 弹窗内改写内容逐字流式吐出，支持 Markdown 格式
- 重新生成或手动输入均计入版本数，累计超过 5 个版本时出 toast 提醒
- **编辑器右侧常驻悬浮菜单**：包含润色一下 / 丰富一下 / 写短一下 / 继续写点 / 调整风格。在未划选文字时，点击悬浮菜单执行**全局改写**（全文润色/扩写/缩写/续写/调整风格）
- 全局改写逐字吐出，加载中悬浮工具栏切换为状态栏，支持点击取消改写
- 用户划选后点击空白处，划选和功能条均消失
- 左侧弹窗改写与右侧对话流采用串行方案，避免左右指令冲突

### 7.4 历史记录

编辑器顶部信息栏提供「历史记录」入口，用户可查看文档的完整修改历史。

**入口状态：**

| 状态 | 表现 |
|------|------|
| 无修改记录 | 图标为默认灰色态，不可点击 |
| 有修改记录 | 图标高亮，可点击 |
| hover | 变为可点击态，提示"查看历史记录" |

**打开历史记录后的界面结构：**

```
┌──────────────────────────────────────────┬────────────────────────┐
│            「还原此版本」（顶部居中）        │   历史版本记录           │
│                                          │                        │
│                                          │   今天                  │
│         文档预览区（只读）                  │   2026-03-30 12:03 [最近更新] │
│                                          │                        │
│         展示选中版本的完整文档内容           │   3月                   │
│                                          │   2026-03-26 11:48     │
│                                          │   2026-03-26 11:31     │
│                                          │   2026-03-26 11:29     │
│                                          │   ...                  │
│                                          │                        │
│                                          │   展开更多 ∨            │
│                                          │                        │
│                                          │   显示更改 [开关]       │
└──────────────────────────────────────────┴────────────────────────┘
```

**交互说明：**

- 点击「历史记录」→ 打开历史版本记录侧面板，**收起右侧对话框**
- 右侧面板按**时间分组**展示（今天 / 本月 / 更早月份），最近更新排在最上方
- 当前版本蓝色高亮，右侧标注「最近更新」标签
- 默认选中当前版本，顶部「还原此版本」按钮置灰
- 选择其他版本后，左侧文档同步展示该版本内容，「还原此版本」变为可点击
- 版本列表底部有「展开更多」按钮，点击加载更早的历史版本
- 面板底部「显示更改」开关：打开后左侧文档展示修改痕迹（增删标记）
- 点击「还原此版本」→ 将文档恢复到选中版本
- 历史版本页面**仅支持查看和还原**，不支持直接编辑
- 被删除/替换的文字仅支持复制操作

## 10. 交互规范

### 10.1 灵感卡片交互

- 默认态：白底灰边 + 关键词 pill 标签
- hover：边框变蓝 + 轻微上浮
- click：选中态蓝底，触发下一步流程
- 每轮 3 张卡片，用户可选择卡片也可直接输入文本（绕过卡片选择）

### 10.2 设定确认卡片交互

- 分组展示（每组有标题 + 多个 label-value 对）
- tag 类型的值以 pill 样式展示
- 底部提示："查看完整设定请点击编辑区"
- 确认按钮：蓝色圆角按钮，点击后进入下一阶段

### 10.3 创作进度条交互

- 垂直方向排列，5 个阶段节点 + 连线
- 已完成阶段：蓝色实心圆 + 蓝色连线
- 当前阶段：蓝色发光 + 放大效果
- 未来阶段：灰色空心圆
- 阶段间连线支持填充动画（跟随进度值平滑过渡）

### 10.4 章节 TOC 交互

- 默认收起，左上角按钮点击展开
- 展开为侧边抽屉（覆盖在编辑器上方）
- 每章显示：状态图标（✅ 已完成 / ⏳ 生成中 / ○ 待生成）+ 章节标题
- 点击章节切换内容，抽屉自动收起

## 11. 关键设计决策

| 决策点 | 方案选择 | 设计考量 |
|--------|---------|---------|
| 电商/知识直接进入 Agent | 跳过模式选择 | 这两个场景用户创作目标明确、流程固定，不需要自由编辑入口 |
| 灵感卡片 3 轮制 | 每个阶段固定 3 轮卡片选择 | 3 轮能充分收集用户偏好，又不至于让用户感到疲倦 |
| 编辑器分阶段展示 | 根据创作阶段切换编辑器内容 | 让用户在对话推进的同时，随时看到当前阶段的结构化成果 |
| Agent/Workflow 双模式 | 小说/剧本场景提供模式选择 | 新手用 Agent 引导从零开始，有经验的创作者用 Workflow 自由编辑 |
| 场景差异化进度标签 | 各场景使用不同的阶段名称 | 同一个 5 阶段结构，用场景化的标签（如"产品→策略→故事线→分幕→精修"）降低用户理解成本 |
| 划词 AI 调整 5 项能力 | 润色/丰富/缩写/续写/氛围增强 | 覆盖正文精修最高频的 5 类需求，同时支持自定义输入 |
